## Current recommended hero

### Main line

### Support copy

처음부터 새로 짜는 일보다 어려운 건, 이미 운영 중인 서비스 안에서 변경 가능한 선을 찾는 일입니다. PHP/CodeIgniter 쇼핑몰 유지보수에서 시작해 문자 발송 서버, 앱 API와 관리자 페이지, AWP 업무 시스템과 3D/BIM 뷰어까지 다뤘습니다. 화면을 고칠 때 API 응답, DB 구조, 권한, 서버와 배포 이후 영향을 같이 봅니다.

### AI workflow line

AI는 판단을 대신하지 않습니다. Cursor, Codex, OpenCode는 낯선 코드의 흐름을 따라가고, 구현안을 비교하고, 검증 목록을 반복하는 시간을 줄이는 도구입니다. 최종 책임은 여전히 사람이 져야 합니다.

## Short version

이미 돌아가는 시스템을 읽고, 필요한 만큼 바꾸는 개발자입니다. PHP/CodeIgniter 유지보수에서 시작해 문자 발송 서버, 앱 API, AWP 업무 시스템과 3D/BIM 뷰어까지 다뤘습니다. AI 도구는 판단을 대신하는 장치가 아니라 검색, 초안, 검증 반복을 줄이는 장치로 사용합니다.

## Professional version

웹 서비스 유지보수와 운영 경험을 바탕으로 화면, API, 데이터베이스, 서버 흐름을 연결해 구현하는 풀스택 개발자입니다. AWP 업무 시스템과 3D/BIM 뷰어, 문자 발송 서버, 앱 API 및 관리자 페이지, 쇼핑몰/웹뷰 앱 유지보수처럼 이미 운영 중인 시스템을 다뤄왔습니다. 최근에는 Cursor, Codex, OpenCode, AGENTS.md, Skills, MCP를 활용해 코드 탐색, 구현안 비교, 문서화, 검증 반복 시간을 줄이는 개발 흐름을 만들고 있습니다.

## Do not use

- “누구에게 따뜻함이 필요한지 살펴봅니다” 같은 감성 문장을 쓰지 않는다.
- “기술과 사람을 연결하는 따뜻한 개발자” 같은 누구에게나 붙는 문장을 쓰지 않는다.
- “새 프로젝트를 처음부터 예쁘게 만드는 일보다”처럼 제작 과정의 방어 논리를 공개 카피에 쓰지 않는다.
- “AI에게 일을 맡기고…”를 메인 소개로 쓰지 않는다. 내부 작업 방식 설명에는 가능하지만, 이력서 기반 포트폴리오 첫 소개로는 추상적이다.
- “Loop Dog Lab”, “댕댕이/멍멍이”를 첫 화면의 주인공으로 쓰지 않는다.
- “귀여운 척을 빼고” 같은 제작 과정 메타 문구를 공개 페이지에 넣지 않는다.
- 검증되지 않은 사이드 프로젝트를 경력보다 먼저 내세우지 않는다.
